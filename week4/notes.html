CH8: Forms

Reset Buttons are considered poor for usability, as they are too easy to erase
the entire form.
form.action = '/an/other.url' used to set the form action attribute so its sent to a
different url.
Pulling user values from form ${input.value} in JS script
placeholder = 'enterSomething here' used to have some information in the form input box before the user enters something
HTML5 .focus() in JS focuses on the element once the page loads.
Form Validation:
Used to make sure the user enters the requested Data in the Form you wanted. HTML5 has an API for basic validation. Custom validation might be required for your purposes.


CH12: Object Oriented Programming

Constructor can be used to create new object using the new operator
Private methods can only be access from within the object.
public methods can be access from outside the object.
getters and setters can be used to access the data from private methods.
Inheritance can be used in JS objects
Mixins allow adding properties and methods from one object to another without inheritance.
Factory Functions can be used with copy(enter object here) to have a function return an object.
Chaining functions: superman.fly().move().xray(); with call all the specified functions.
ES6 arrow functions do not have their over .this context, .this remains bound to the original object.
ES6 allows for Modules to be used
use import {} from './your.js' to import another JS file.
use export {function1, function2} to export your functions.
import {function1, function2} from './yourJSfile'; to import specific functions from another JS file.
import * as nameThis from './yourJSfile'; imports all values and functions from the JS file. nameThis.function({2,6,10}); allows you to put in your of vales into the functions with the namespace of nameThis
single/multiple functions and values can be exported. objects can also be exported.
Do not have more than one default export, the result would be a syntax error.
MVC:
models: objects the implement functionality for creating,reading,updating and deleting (known as CRUD tasks).
Views: provide visual representation of the model showing all the relevant information. In web app this would be
the HTML displayed on the web page.
Controllers: link models and views together by communicating between them. Controllers respond to events, process the data
and update the model and view.
MV* with JS rather than MVC since JS does not strictly follow the controller pattern.
MV* Frameworks:
Aurelia which uses ES6
Angular - created by google. Uses ng- attributes.
Vue - open source framework that takes the best of React & Angular
Templates: MVC frameworks use templating to insert dynamic data into the pages.

Hello {{ name }}

uses templating to insert a name. Some frameworks use handlebars {{}},
EJS uses <%= name =>
